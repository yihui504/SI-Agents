## 6. 移植建议

### 高优先级
1. **实现任务源**：移植 SkVM 的 task-source.ts 逻辑，支持优化任务来源管理
2. **集成 NanobotPolicy**：将新实现的执行命令策略集成到优化流程

### 中优先级
1. **移植 runtime 模块**：实现运行时调度功能
2. **完善 profiler**：移植性能分析能力

### 低优先级
1. **缓存管理**：实现技能执行结果缓存
2. **特定场景优化器**：针对特定语言和场景的优化

## 7. v0.2.0 改进记录

### 7.1 功能移植与修复

| 改进项 | 描述 |
|--------|------|
| OptimizationLoop 安全验证修复 | 移除占位符实现，补全安全验证逻辑 |
| SkillOptimizer 支持内置 HeadlessAgent | 不再硬依赖 opencode CLI，可使用内置 Agent 执行优化 |
| 策略工具函数提取到 policy-utils.ts | 消除约 600 行重复代码，统一策略辅助函数 |
| ESM 兼容性修复 | 将 `require()` 调用替换为 `import`，确保 ESM 环境正常运行 |
| 可配置规则详情 URL | 规则详情链接支持通过配置自定义，不再硬编码 |

### 7.2 可观测性与监控

| 改进项 | 描述 |
|--------|------|
| LangfuseClient 批量发送 + 重试 + 并发控制 | 提升遥测数据上报效率与可靠性 |
| 流式响应增量策略检查 | 先查后发模式，流式输出时逐段进行策略合规检查 |
| 健康检查与监控端点增强 | 扩展健康检查指标，提供更丰富的运行状态信息 |
| 结构化审计日志 | 审计事件采用结构化格式输出，便于检索与分析 |
| OpenTelemetry 兼容追踪 | 支持标准 OTel 协议的分布式追踪集成 |

### 7.3 成本与资源管理

| 改进项 | 描述 |
|--------|------|
| BareAgentAdapter 成本估算改进 | 支持可配置价格表，内置 GLM 实际价格，估算更准确 |
| LangfuseClient LRU 内存管理 | 限制遥测数据内存占用，防止长时间运行内存泄漏 |
| EFSM 快照缓存 | 缓存状态机快照，减少重复计算开销 |

### 7.4 安全增强

| 改进项 | 描述 |
|--------|------|
| SSRF 防护 | 防止服务端请求伪造攻击，校验出站请求目标 |
| 确认流程安全增强 | 加强确认机制的安全性，防止绕过确认直接执行 |
| NanobotPolicy ReDoS 防护 | 修复正则表达式拒绝服务漏洞，限制回溯深度 |
| 规则文件 fail-closed | 规则文件加载失败时默认拒绝，而非放行 |
| 密钥安全引用 | 敏感密钥通过安全引用机制访问，避免明文暴露 |

### 7.5 性能与正确性

| 改进项 | 描述 |
|--------|------|
| 污点传播 O(n²)→O(n) | 优化污点分析算法，将时间复杂度从二次降为线性 |
| 并发安全（Mutex + Promise Chain） | 引入互斥锁与 Promise 链，确保并发操作安全 |
| Zod 边界验证 | 使用 Zod schema 对所有外部输入进行边界校验 |
| 策略注册排序 | 策略按优先级排序注册，保证执行顺序确定性 |

### 7.6 API 与扩展性

| 改进项 | 描述 |
|--------|------|
| Agent 工具注册 API | 提供标准接口供 Agent 动态注册工具 |
| 文件持久化存储 | 支持将运行数据持久化到文件系统 |

## 8. 当前状态总结

- **已移植功能**：优化器框架、优化循环（含安全验证修复）、工作区管理、运行状态、安全扫描、约束注入、验证器
- **未移植功能**：任务源、runtime、完整 profiler
- **新增功能**：完整的安全策略系统（来自 ArbiterOS）、v0.2.0 全部改进（见第 7 节）
- **测试覆盖**：基础功能与优化循环有测试，任务源、runtime 等缺失功能无测试
